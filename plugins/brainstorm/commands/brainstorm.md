Run the `brainstorm` skill now.

- `$ARGUMENTS` names the seed (e.g. `/brainstorm what should I build next for my clinic
  workflow`) → open the session with it.
- `$ARGUMENTS` empty → ask what the brainstorm is for, then proceed.

The skill owns the whole procedure — ground → diverge → converge → route (and, for a
picked learning idea, Learn-hub stubs → `/sync-vault` handoff). This command is only the
manual trigger for it.
